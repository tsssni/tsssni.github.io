今天正式开始开发metatron-renderer, 名字由来第一版发布再说.

工具链
#

原本计划使用c++20 modules, 因为懒得分离定义和实现. linux上只有clang对模块有正式支持, 但当clang的exe为符号链接时模块无法找到系统头文件, 而NixOS更是封装了一层bash. 官方推荐手动设置-I/path/to/system/headers, 但最后clang会引用到gcc的头文件, 导致找不到定义. 即使不用模块, 现在nixos-24.11的clang也有封装问题, 同样找不到系统头文件. 之后试过用c++23 import std;, 但cmake即使打开实验特性也会报only support libc++.

rust也是个选择, 但rust的trait和impl其实也一定程度分离了定义和实现. 加上我对c++还是有好感的, 也希望项目里能练习一些新特性, 最后还是fallback到更稳妥的gcc+headers

编辑器
#

使用自己配置的nixvim, 测试模块时发现clangd默认不支持c++20语法特性, 需要通过.clangd文件设置.

CompileFlags:
  Add: [-std=c++20]

构建系统
#

为了兼容性使用CMake, 构建流程的设计目标如下:

  • 模块编译为动态库
  • 自动分析依赖关系
  • 第三方依赖可配置

项目结构
#

整体结构分为以下处理单元集合:

  • lib: 第三方依赖
  • src: 项目内模块
  • exe: 可执行程序

此外还有额外的文件夹:

  • cmake: 自定义的cmake函数

分批处理
#

将lib/src/exe下面的子文件作为最小处理单元, metatron_classify负责收集处理单元. path是单元集合目录, 目录下的单元会使用相似的构建方式, 例如exe下的单元都会通过add_executable创建target. mode记录当前处理模式, 同样可设置为"lib"|"src"|"exe", 主要用于单元构建时选择不同的策略. 为防止冲突模块名会添加metatron-的前缀.

function(metatron_classify path mode)
  if (IS_DIRECTORY ${path})
    file(GLOB units RELATIVE ${path} ${path}/*)
    foreach(unit ${units})
      if(IS_DIRECTORY ${path}/${unit})
        metatron_evaluate(metatron-${unit} ${path}/${unit} ${mode})
      endif()
    endforeach()
  endif()
endfunction()

单元设置
#

每个单元下都可以通过setup.cmake做一些设置, 例如exe/renderer/setup.cmake, 大部分工作都在这里完成, CMakeLists.txt只做简单的配置与全局变量定义.

cmake_minimum_required(VERSION 3.30.5)

project(metatron-renderer LANGUAGES CXX)
set(CMAKE_CXX_STANDARD 20)

set(metatron-root ${CMAKE_CURRENT_LIST_DIR})
set(metatron-src ${metatron-root}/src)
set(metatron-bin ${CMAKE_BINARY_DIR}/bin)
set(metatron-lib ${CMAKE_BINARY_DIR}/lib)
set(metatron-gen ${CMAKE_BINARY_DIR}/gen)

include(cmake/evaluate.cmake)
include(lib/setup.cmake)
include(src/setup.cmake)
include(exe/setup.cmake)

对于处理单元集合, setup.cmake只是调用metatron_classify.

message(STATUS "processing libraries ...")
metatron_classify("${metatron-root}/src" "src")

对于exe/src, setup.cmake主要用于分析依赖关系, 将依赖的模块存储在metatron-deps中.

list(APPEND metatron-deps mimalloc)

对于lib, setup.cmake还会设置第三方库的构建选项. 第三方库也需要设置metatron-deps, 主要用于以统一的格式设置依赖.

set(MI_USE_CXX ON CACHE BOOL "")
set(MI_BUILD_STATIC OFF CACHE BOOL "")
set(MI_BUILD_OBJECT OFF CACHE BOOL "")
set(MI_BUILD_TESTS OFF CACHE BOOL "")
add_subdirectory(${path}/mimalloc)
list(APPEND metatron-deps mimalloc)

求解模块
#

模块生成过程在metatron_evaluate中.

对于lib, 创建interface target, 用于传递与第三方库的依赖关系, 并不会生成实际构建结果.

message(STATUS "build 3rd-party library ${target}")
add_library(${target} INTERFACE)

src/exe下如果有cpp文件, 则创建target, 最后会生成动态库或可执行文件.

if(${mode} STREQUAL "exe")
    message(STATUS "build executable ${target}")
    add_executable(${target} ${sources})
else()
    message(STATUS "build library ${target}")
    add_library(${target} SHARED EXCLUDE_FROM_ALL ${sources})
endif()

src下如果只有hpp文件, 纯头文件库会采用和lib类似的处理方式, 因为只需要被include.

message(STATUS "build header-only library ${target}")
add_library(${target} INTERFACE)
set(mode "lib")

执行setup.cmake, 收集依赖项并链接. src下的源码都必须位于metatron文件夹下, 例如src/test/metatron/test.hpp, 这样头文件都可以通过#include <metatron/...>引入, 且分析完依赖关系后, 只有依赖的模块的头文件会被补全.

# execute setup file
set(metatron-deps)
set(setup-file ${path}/setup.cmake)
if(EXISTS ${setup-file})
    include(${setup-file})
endif()

# solve dependencies
set(linked-libs)
set(included-libs)
foreach(dep ${metatron-deps})
    if(${mode} STREQUAL "lib")
        list(APPEND linked-libs ${dep})
    else()
        list(APPEND linked-libs metatron-${dep})
        list(APPEND included-libs ${metatron-src}/${dep})
    endif()
endforeach()

# link dependencies
if(linked-libs)
    message(STATUS "${target} link libraries ${linked-libs}")
    if(${mode} STREQUAL "lib")
        target_link_libraries(${target} INTERFACE ${linked-libs})
    else()
        target_link_libraries(${target} PUBLIC ${linked-libs})
        target_include_directories(${target} PUBLIC ${included-libs})
    endif()
endif()

构建运行
#

为避免cmake命令行参数不一致导致的问题, 项目中含有CMakePresets.json.

{
  "cmakeMinimumRequired": {
    "major": 3,
    "minor": 30,
    "patch": 5
  },
  "configurePresets": [
    {
      "binaryDir": "${sourceDir}/build/${presetName}",
      "cacheVariables": {
        "CMAKE_BUILD_TYPE": "Debug",
        "CMAKE_EXPORT_COMPILE_COMMANDS": "ON"
      },
      "name": "linux-native-dev",
      "generator": "Ninja"
    },
    {
      "binaryDir": "${sourceDir}/build/${presetName}",
      "cacheVariables": {
        "CMAKE_BUILD_TYPE": "Release",
        "CMAKE_EXPORT_COMPILE_COMMANDS": "ON"
      },
      "name": "linux-native-rel",
      "generator": "Ninja"
    }
  ],
  "version": 8
}

运行cmake -B build --preset linux-native-dev即可生成构建目标, cmake --build build启动构建.
